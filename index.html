RPA Cyber Cover — Police CyberAlarm transition period

Police CyberAlarm have announced that new installations of the data collector are currently paused during a planned transition period. The RPA's cyber cover condition is registration with PCA — which has historically been possible without installing the collector — but the practical position during this transition is best confirmed directly. If your school is not already registered, contact the RPA at RPA.DFE@education.gov.uk to confirm how cyber cover can be approved while the PCA service is in transition. Do not assume cyber cover is unavailable until you have spoken to them.

The other three RPA cyber cover conditions remain in effect: offline backups, annual NCSC cyber security training for all staff and governors with system access, and a documented Cyber Response Plan (which this tool helps you build).

  • Fill-in-the-blanks templates with bracketed placeholders for school-specific details — designed to force school-specific thinking rather than producing generic boilerplate

Communications templates

  • Sample drafts for parent letters, staff briefings, ICO notifications, governor briefings and the website holding statement
  • "Use this draft as a starting point" button for one-click insertion

Governor / Trustee Report

  • Plain-English one-page printable summary — readiness % score, RAG breakdown and an overall verdict that adapts to the school's score
  • "How do you assure yourselves..." questions for each non-green readiness area, supporting Ofsted-style inspection assurance questioning
  • Suggested wording for governor meeting minutes — minuting evidences cyber assurance during inspection
  • For SLT / For Governors split in the Recommended Next Steps

Sector context for 2026

  • Banner noting the Police CyberAlarm transition period — new installations of the data collector are currently paused. About the Cyber Incident Response Planner

A free planning tool for UK schools and colleges — helping schools build, test and maintain a working Cyber Incident Response Plan, mapped to NCSC, DfE Digital Standards 2030, DfE Risk Protection Arrangement (RPA) and the Ofsted Inspection Toolkit.

✅ What this tool does

  • 12-question readiness check with RAG scoring against four frameworks
  • 10-section Plan Builder for a working incident response plan
  • Six pre-populated playbooks (ransomware, data breach, account compromise, phishing, denial of service, insider threat)
  • Communication templates with sample drafts for parents, staff, ICO and governors
  • One-page Governor Report supporting Ofsted assurance questioning
  • Sticky table of contents on the generated plan for easy navigation
  • Save and reload your progress as a local JSON file

⚠️ What this tool is not

  • Not a substitute for NCSC's official incident management guidance
  • Not legal, regulatory or insurance advice
  • Not a guarantee of compliance with any framework
  • Not a replacement for an external incident response retainer
  • Not affiliated with the DfE, NCSC, RPA, Ofsted or any government body

Always validate the final plan with your IT support, Data Protection Officer, SLT and governing body before relying on it.

👥 Who is this for?

Any UK state, independent or further education setting that needs to write or review a cyber incident response plan, including:

  • Senior leadership team digital leads
  • Network managers and in-house IT support staff
  • Multi-academy trust IT directors
  • Business managers and bursars working on RPA renewals
  • Governors and trustees seeking assurance on cyber resilience
  • Data Protection Officers reviewing breach response readiness

📚 Frameworks and standards mapped

Each readiness question and plan section is explicitly tagged to the framework(s) it maps against, so schools always know which expectation each part addresses.

NCSC INCIDENT MANAGEMENT

The structure of the plan — key contacts, severity grading, categorisation, escalation, playbooks, post-incident review — follows the NCSC's published incident management guidance for organisations of any size.

Read the source guidance →
DFE DIGITAL STANDARDS 2030 — CYBER SECURITY CORE STANDARD

Readiness questions reflect the DfE's stated expectations on risk assessment, staff training, MFA, response planning, asset registers and patching. One of the six core Digital and Technology Standards UK schools should be working towards by 2030.

Read the source guidance →
DFE RISK PROTECTION ARRANGEMENT (RPA) — CYBER RESPONSE PLAN

Plan structure aligns with the impact assessment, communication templates and recovery event recording required by RPA cyber cover. Schools using RPA can use the generated plan as the working document for their cyber response.

2026 note: Police CyberAlarm has paused new installations of the data collector during a transition period. The RPA condition is PCA registration, which has historically been possible without installing the collector — but schools should contact RPA.DFE@education.gov.uk to confirm the practical position during the transition.

Read the source guidance →
OFSTED INSPECTION TOOLKIT (NOVEMBER 2025)

Ofsted does not directly inspect cyber security, but several aspects of cyber response intersect with the safeguarding evaluation (Met / Not Met) and leadership and management. The Governor Report uses the Ofsted-style "How do you assure yourselves..." framing for assurance questioning.

Read the source guidance →

🔒 Privacy

  • No server. The tool runs entirely in your browser.
  • Your plan data never leaves your device. Held in your browser session and wiped on tab close.
  • Save to file. Progress is saved as a JSON file directly to your device. The author cannot see it.
  • Anonymous page-view counts only. A privacy-friendly analytics service (GoatCounter) records that someone visited a page. No cookies, no fingerprinting, no advertising trackers, GDPR-compliant.
  • Open source. Anyone can review the source code on GitHub.

🛡️ Security posture

  • Content Security Policy locks down remote content to a single trusted analytics endpoint — nothing else can load
  • JSON imports validated against schema (blocks prototype pollution and type confusion)
  • All user input HTML-escaped before rendering
  • External links use rel="noopener noreferrer"
  • Automated security test suite available in the repo

🤝 Built for the school IT community

Built and maintained by Christopher Maddocks, a former ANME (Association of Network Managers in Education) Ambassador, as a free contribution to the UK education community.

It's a companion to the DfE Digital Standards 2030 — Self-Assessment Tool, focusing specifically on the cyber response and recovery layer where the Self-Assessment Tool's Cyber Security standard leaves off.

If you have feedback, find a problem, or want to contribute improvements, the easiest route is the 2-minute feedback form. The source code lives on GitHub at github.com/cmaddocks-uk/cyber-response if you'd prefer to raise an issue or pull request directly.
